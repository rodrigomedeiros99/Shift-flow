rotation_rules

Stores configurable fairness rules.

rotation_rules
Column	Type
id	uuid
facility_id	uuid
department_id	uuid
shift_key_id	uuid
task_type_id	uuid
lookback_days	integer
warning_threshold	integer
active	boolean
created_at	timestamptz

Example:

Task: CL
Lookback: 4 days
Warning threshold: 2

Meaning:

If John was planned for CL 2+ times in the last 4 planned shifts,
show a rotation notice.

Unique constraint

Only one active/published plan per:

facility + department + shift_key + plan_date

This prevents duplicate plans.

RLS high-level rules
Admin can access everything in their facility.
Supervisor can view/edit operational plans in their facility.
Inbound Leader can create/edit inbound plans only.
Outbound Leader can create/edit outbound plans only.
Viewer can only read published plans and TV mode data.
No user can access another facility unless authorized.
