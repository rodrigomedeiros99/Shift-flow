## Development Roadmap v1.0

## Purpose

This roadmap defines the build order for ShiftFlow DFC 5523.

The project must be developed phase by phase.

Claude Code, Codex, Cursor, or any AI coding agent must not attempt to build the entire application at once.

Each phase must be completed, tested, and verified before moving to the next phase.

---

# Phase 1 — Project Foundation

## Goal

Create the technical foundation of the application.

## Build

* Next.js latest stable with App Router
* TypeScript strict mode
* TailwindCSS
* ESLint
* Prettier
* Supabase client setup
* Environment variable structure
* Base folder structure
* App layout
* Dashboard shell
* Sidebar navigation
* Header navigation
* Loading states
* Error states
* Empty states

## Routes

* `/login`
* `/dashboard`
* `/settings`

## Done When

* App runs with `npm run dev`
* App builds with `npm run build`
* No TypeScript errors
* No `any`
* Layout is responsive
* Vercel-compatible setup exists

# Phase 7 — Fair Rotation Recommendation Engine

## Goal

Add fair planning intelligence.

## Build

* Planned Assignment History lookup
* Rotation recommendations
* Assignment frequency calculation
* Rotation notices
* Leader override flow
* Rotation score
* Auto-plan improvements

## Rules

* Use Planned Assignment History for fairness
* Do not use Activity History as primary fairness source
* Never block the leader
* Warn and recommend only
* Leadership can assign anyway

## Done When

* System warns if someone recently performed the same planned task
* System recommends better alternatives
* Leader can override recommendation
* Auto-plan tries to rotate people fairly

# Build Rule

Do not skip phases.

Do not build future phases before the current phase is stable.

Do not create placeholder implementations.

Do not hardcode operational values.

Every phase must produce production-quality code.
